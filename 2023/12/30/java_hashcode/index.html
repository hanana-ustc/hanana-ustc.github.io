Java hashcode到底从何而来？

众说纷纭的hashcode

hashcode作为Object类下重要的本地方法，在HashMap等一众Hash集合类中发挥巨大作用。但关于它的具体实现，网络上的各种博客简直是众说纷纭。最常见到的一种说法称，hashcode方法返回的是对象在JVM中的地址值。但笔者认为这种说法似乎站不住脚，首先若所有对象hashcode都与JVM中地址挂钩，那被JIT编译器捕获的进行栈上分配而被标量替换的对象该如何生成对应的哈希值呢？似乎光用JVM中的地址值生成hashcode并不是那么靠谱。

接下来将逐个分析各种实现方式。

0. 随机数生成hashcode

  • 策略: 使用全局 Park-Miller 随机数生成器。
  • 问题: 多个线程可能会生成相同的随机数，且在多处理器系统中可能会产生大量的缓存一致性开销。

1. 对象地址和全局随机数进行异或运算生成hashcode

  • 策略: 使用对象地址和全局随机数进行异或运算。
  • 特点: 在 STW（Stop-The-World）操作之间是稳定的，适用于一些同步方案。

2. 直接返回1作为hashcode

  • 策略: 返回常数值 1，用于敏感性测试。

3. 使用全局递增序号作为hashcode

  • 策略: 使用全局递增的序列号。

4. 对象地址作为hashcode

  • 策略: 直接使用对象的地址。

5. 使用Marsaglia 的 xor-shift 随机数生成器生成hashcode

  • 策略: 使用 Marsaglia 的 xor-shift 随机数生成器，每个线程有独立的随机状态。
  • 特点: 可能是未来版本的默认实现，具有良好的随机性和性能。

为什么是你？xor-shift随机数生成器

Marsaglia 的 xor-shift 随机数生成器因其高效性和良好的随机性，成为 hashCode 生成的一个受欢迎的选择。

Marsaglia 的 xor-shift 随机数生成器是一种高效的伪随机数生成算法，基于位操作（主要是异或和位移）来生成随机数。这个算法由知名数学家 George Marsaglia 提出，旨在提供一种简单而快速的随机数生成方法。

简单的xor-shift随机数生成器实现原理：

  1. 状态表示：

    • xor-shift 算法使用一个整数（通常是 32 位或 64 位）作为内部状态。这一状态存储了生成随机数所需的信息。

  2. 随机数生成过程：

    • 随机数生成的基本步骤是：

      • 通过一系列的位运算（异或和位移），更新内部状态。
      • 返回更新后的状态作为产生的伪随机数。

  3. 参数选择：

    • 所有的参数（即位移量）都旨在通过不同的位移方式来确保随机性，避免生成序列中的模式。

  4. 周期性：

    • xor-shift 算法具有很大的周期（在 32 位情况下，可以达到 2^32 - 1），即在生成一定数量的随机数后，生成的序列会重复。因此，提出合适的初始状态（种子）尤为重要，以确保随机性。

首先看看其他4种生成方式的优缺点。

  1. 全局随机数生成器：
    • 优点：简单，易于实现。
    • 缺点：可能引入竞争条件，多线程中同一时刻会生成相同的随机数，降低分布均匀性和应用效果。
  2. 稳态哈希生成（基于地址）：
    • 优点：在 STW 操作间保持稳定，可重用。
    • 缺点：可能导致哈希码值的分布不均匀，尤其是在相邻对象之间，可能增加哈希冲突。
  3. 常量哈希（返回 1 或递增序列号）：
    • 优点：在特定测试中便于使用，简单。
    • 缺点：生成的哈希值缺乏多样性，几乎总是相同，绝对不适合实际应用中的哈希表。
  4. 直接使用地址：
    • 优点：简单直接，容易实现。
    • 缺点：不考虑随机性，可能导致相近内存位置的对象产生相同的哈希值，从而增加哈希冲突的概率。

ps：其实它们的缺点都指向一个问题——哈希冲突！

而xor-shift随机数生成器

  1. 高效性：
    • xor-shift 算法非常快，因为它只依赖于位运算（异或和移位）。这使得它在生成哈希码时不会引入太多的性能开销。
  2. 简单实现：
    • xor-shift 的实现相对简单，代码长度较少，易于维护和理解。
  3. 良好的随机性：
    • 这种随机数生成器能够生成具有较长周期的随机数序列，因此在较长时间内使用时仍然可以保持随机性，降低碰撞的可能性。

ps：可以看到xor-shift随机数生成器拥有更长的周期，相比其他方案哈希冲突可能性更小。
