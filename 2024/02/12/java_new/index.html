java8关键特性

  • lambda表达式
    lambda表达式为接口提供一个方法用来实现。

    Lambda表达式的具体规范：

    • 标准格式为：([参数类型 参数名称,]…) ‐> { 代码语句，包括返回值 }
    • 和匿名内部类不同，Lambda仅支持接口，不支持抽象类
    • 接口内部必须有且仅有一个抽象方法（可以有多个方法，但是必须保证其他方法有默认实现，必须留一个抽象方法出来）

    对于一个仅有一个方法需要实现的接口，可以使用lambda表达式实现这个接口。

    接口的实现除了自己手动实现外，若其他类中有相同作用的方法能够实现接口可以直接进行引用。

    标准格式为：类名::方法名 //静态方法

java9关键特性

  • 模块机制

    Java 9 引入了一个重大的新特性，即模块化机制，它被称为 Java Platform Module System（JPMS）。模块化机制的目的是解决 Java 应用程序的可扩展性和复杂性问题。它大大减少了应用程序依赖库的大小。

    在 Java 9 中，模块由 module-info.java 文件定义，其中包含模块的名称、依赖项和导出的公共接口。

    采用模块机制后，我们不能通过反射机制去获取其他模块信息。若想开放反射机制给某些模块，则需要添加open关键字：

  • JShell交互式编程

    JShell 是 Java 9 引入的一个交互式编程工具，它为开发人员提供了一个即时执行代码的环境，无需创建和编译完整的 Java 程序。它可以在命令行中直接输入和执行 Java 代码片段，并立即返回结果。所用语法仍然为java语法。(ps:好像解释性语言，但又不那么解释性。)

java17关键特性

  • 密封类型

  • 密封类型可以说是Java 17正式推出的又一重磅类型，它在Java 15首次提出并测试了两个版本。

    在Java中，我们可以通过继承（extends关键字）来实现类的能力复用、扩展与增强。但有的时候，可能并不是所有的类我们都希望能够被继承。所以，我们需要对继承关系有一些限制的控制手段，而密封类的作用就是限制类的继承。

    在没有密封类型前我们可以使用final关键字限制类的继承:

    但是使用final关键字限制的类都不能进行继承，包括自己想能继承的类。要解决这个问题，密封类型孕育而生:

    密封类型有以下要求：

    • 可以基于普通类、抽象类、接口，也可以是继承自其他接抽象类的子类或是实现其他接口的类等。
    • 必须有子类继承，且不能是匿名内部类或是lambda的形式。
    • sealed写在原来final的位置，但是不能和final、non-sealed关键字同时出现，只能选择其一。
    • 继承的子类必须显式标记为final、sealed或是non-sealed类型。
