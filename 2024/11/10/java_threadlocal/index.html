先说结论，ThreadLocal 对于基本数据类型可以解决线程安全问题，但对于引用类型来说则并不能解决线程安全问题。

大部分博客或者面试题都说 ThreadLocal 实现了线程间的变量隔离。这句话本身是没有任何问题的，ThreadLocal是通过在每个线程中都存在一个ThreadLocalMap来实现线程间的变量隔离，显然这对基本数据类型是线程安全的，但它对引用数据类型来说却是线程不安全的。

可以看到主线程中的值被子线程修改了，ThreadLocal 并未实现对象线程间的隔离，存在线程安全问题。

这是由于 ThreadLocal 本质上 set 的值是对象的地址值，地址值是线程隔离的，但是地址值所指向的对象都是堆中的同一个，我们对子线程的值所指向的对象进行修改，其实是对堆中的对象进行修改，因此会影响主线程中 ThreadLocal value 值所指向的对象。
